##### **Changements cles de la lettre BMF 2024 (Facturation electronique)**

**1. Classification des erreurs**

* Erreurs de format : Se produisent lorsque les specifications techniques ne sont pas respectees. Le document est alors retrograde au rang de "autre facture" et doit etre remplace par une nouvelle facture electronique.
* Erreurs de regles metier : Concernent les dependances logiques ou les donnees obligatoires incompletes. Le document reste une facture electronique mais necessite une correction formelle des informations obligatoires specifiques.
* Validation : Les deux peuvent etre verifiees via des applications de validation. Une verification reussie (avec un rapport sauvegarde) protege l'entrepreneur au titre des regles de "diligence raisonnable". Les erreurs de contenu (par exemple, une mauvaise description du service) necessitent toujours un examen manuel.

**2. Donnees structurees obligatoires**

* Pas de references externes : Toutes les informations obligatoires relatives a la TVA doivent figurer dans la partie structuree du fichier.
* Pieces jointes : Les supplements sont autorises sous forme de fichiers integres (pas de liens), mais ils ne peuvent pas remplacer les donnees obligatoires.
* Rigueur : Les references a des contrats externes ou des bons de livraison pour les dates de prestation ne sont generalement plus suffisantes. Meme les formules courantes comme "date de prestation egale date de facture" doivent desormais faire partie des donnees structurees.

**3. Types de factures et consentement**

* Petits montants/Titres de transport : La facturation electronique n'est pas obligatoire mais autorisee sans le consentement du destinataire.
* Autres formats electroniques : Pour tout format qui n'est pas une facture electronique standard (par exemple, un simple PDF), le consentement du destinataire est toujours requis.

**4. Corrections et cas specifiques**

* Modifications de service : Necessitent une correction de facture ou un avoir specifique faisant reference a la facture originale. Les simples reductions de prix ne necessitent pas forcement une correction.
* Exonerations fiscales : Si une exoneration de TVA est levee (option pour la taxation), l'obligation d'emettre une facture electronique structuree s'applique.

##### **Resume**

**Classification des erreurs** : Le BMF distingue les erreurs de format (defaillances techniques de syntaxe) des erreurs de regles metier (incoherences logiques). Les erreurs de format disqualifient un document en tant que facture electronique, necessitant un remplacement complet. Les erreurs de regles metier ne necessitent generalement qu'une correction des donnees specifiques.

**Validation** : La validation technique est essentielle mais ne remplace pas un examen substantiel du contenu. Il est conseille de conserver les rapports de validation comme preuve de diligence raisonnable.

**Exigence de donnees structurees** : Toutes les informations obligatoires relatives a la TVA doivent etre contenues dans la partie structuree de la facture electronique. Les liens externes ou les references aux pieces jointes (par exemple, les bons de livraison pour la date de prestation) ne sont plus suffisants pour les details obligatoires.

**Consentement** : Alors que les factures electroniques standard peuvent etre emises sans le consentement du destinataire dans les cas B2B, les "autres" formats electroniques (comme les simples PDF) necessitent toujours l'accord du destinataire.

**Exonerations fiscales** : Si une exoneration de TVA est levee (conformement a l'art. 9 par. 1 UStG), l'obligation d'emettre une facture electronique structuree reste en vigueur.
